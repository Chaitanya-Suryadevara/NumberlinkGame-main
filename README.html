Numberlink React Native Game - Technical Documentation

1. Project Overview

This project is a 5x5 Numberlink-style puzzle game built using React Native and TypeScript. The player connects matching colored endpoints by dragging paths across the board. The puzzle is complete when all matching pairs are connected and every cell in the grid is filled.

3. Main Screens

3.1 App.tsx

App.tsx controls the main navigation flow. It decides whether to show the Home screen, Name screen, or Game screen. 

import React, {useState} from 'react';
import {StatusBar} from 'react-native';
import {HomeScreen} from './src/screens/HomeScreen';
import {NameScreen} from './src/screens/NameScreen';
import {GameScreen} from './src/screens/GameScreen';

type AppStep = 'home' | 'name' | 'game';

function App(): React.JSX.Element {
  // Controls which screen is currently visible
  const [step, setStep] = useState<AppStep>('home');

  // Stores player name entered in NameScreen
  const [playerName, setPlayerName] = useState('');

  return (
    <>
      <StatusBar barStyle="light-content" />

      {/* First screen: animated home screen */}
      {step === 'home' && (
        <HomeScreen onStart={() => setStep('name')} />
      )}

      {/* Second screen: player enters name */}
      {step === 'name' && (
        <NameScreen
          onContinue={name => {
            setPlayerName(name);
            setStep('game');
          }}
        />
      )}

      {/* Third screen: actual puzzle game */}
      {step === 'game' && (
        <GameScreen playerName={playerName} />
      )}
    </>
  );
}

export default App;

10. How API is Used in GameScreen

10.1 Loading Levels

const loadInitialData = async () => {
  try {
    setLoading(true);
    setApiStatus('GET /levels loading...');

    // Calls API to get levels
    const apiLevels = await fetchLevels();

    // If API has levels, use them. Otherwise use sample level.
    const validLevels = apiLevels.length > 0 ? apiLevels : [sampleLevel];

    // Apply random color/letter design
    const redesignedLevels = validLevels.map(item => redesignLevel(item));

    setLevels(redesignedLevels);

    const firstLevel = redesignedLevels[0];
    setLevel(firstLevel);

    // Load scores for first level
    await loadScores(firstLevel.id);

    resetRoundState();
    setApiOnline(true);
  } catch (error) {
    // If API fails, fallback to local sample
    const local = redesignLevel(sampleLevel);

    setApiOnline(false);
    setLevels([local]);
    setLevel(local);
    resetRoundState();
    setStatusMessage('API is offline. Using local sample level.');
  } finally {
    setLoading(false);
  }
};

10.2 Loading Scores

const loadScores = async (levelId: string) => {
  try {
    setApiStatus(`GET /scores?levelId=${levelId} loading...`);

    // Fetch scores for current level
    const apiScores = await fetchScores(levelId);

    // Current level scoreboard
    setScores(apiScores);

    // Add scores into all-level scoreboard
    addToAllScores(apiScores);

    setApiStatus(`GET /scores success. Scores: ${apiScores.length}`);
  } catch (error) {
    setApiStatus('GET /scores failed');
  }
};

10.3 Submitting Score

const handleSubmitScore = async () => {
  // Do not submit if puzzle is incomplete
  if (!isWin(level, paths)) {
    setStatusMessage(
      'Game is not complete. Connect all matching colors and fill every grid cell before submitting.',
    );
    return;
  }

  // Prevent duplicate score submission
  if (scoreSubmitted) {
    setStatusMessage('Score already submitted for this level attempt.');
    return;
  }

  const newScore: Score = {
    id: `local-${level.id}-${Date.now()}`,
    levelId: level.id,
    playerName: playerName || 'Player',
    timeSeconds: seconds,
    createdAt: new Date().toISOString(),
  };

  // Optimistic UI update
  setScoreSubmitted(true);
  setHasWon(true);
  setScores(prev => [newScore, ...prev]);
  addToAllScores([newScore]);

  try {
    // Submit to API
    await submitScore(newScore);

    setStatusMessage('Score submitted successfully.');

    // Reload API scores for current level
    await loadScores(level.id);
  } catch (error) {
    // Local score remains visible even if API fails
    setStatusMessage('Score saved locally, but API submission failed.');
  }
};

12. Solver Approach

The solver is independent from the React UI. It accepts a level and returns either a full solution or null. 

solveLevel(level: Level): Solution | null

The solver uses depth-first search and backtracking.

  • It starts from one endpoint.
  • It checks neighboring cells in four directions.
  • It avoids used cells.
  • It rejects invalid paths.
  • It backtracks when a path cannot complete.
  • It returns null if the board is not solvable.

Hint Usage

const handleHint = () => {
  const solution = solveLevel(level);

  if (!solution) {
    setStatusMessage('This board is unsolvable.');
    return;
  }

  // Finds next valid point from solver solution
  for (const pair of level.pairs) {
    const solvedPath = solution.paths[pair.id];
    const currentPath = paths[pair.id] || [pair.start];

    const nextPoint = solvedPath[currentPath.length];

    if (nextPoint) {
      setPaths(prev => ({
        ...prev,
        [pair.id]: [...currentPath, nextPoint],
      }));

      setStatusMessage(
        `Hint: Continue ${pair.id} to row ${nextPoint.y + 1}, column ${nextPoint.x + 1}.`,
      );
      return;
    }
  }
};

13. Board Interaction

Board handles drag logic. When a player touches an endpoint, a path starts. As the player drags across adjacent cells, the path grows. 

Board.tsx
  ├── Detect touch start
  ├── Identify selected endpoint
  ├── Track active pair
  ├── Add valid adjacent cells
  ├── Reject overlap
  ├── Reject diagonal movement
  └── Update paths in GameScreen

14. Win Condition

The win condition is checked inside boardUtils.ts. 

isWin(level, paths)

The function returns true only when:

  • Every pair has a path.
  • Every path starts and ends at matching endpoints.
  • All movements are adjacent.
  • All 25 grid cells are filled.

16. Reset and New Design

Reset does more than clear the board. It also changes the visual style. The level keeps the same layout, but colors and alphabets change. 

const resetGameWithNewDesign = () => {
  const redesigned = redesignLevel(level);

  setLevel(redesigned);

  setLevels(prev =>
    prev.map(item => (item.id === level.id ? redesigned : item)),
  );

  resetRoundState();

  setStatusMessage('New design loaded. Alphabets and colors changed.');
};

17. Redesign Logic

function redesignLevel(level: Level): Level {
  const palette =
    colorPalettes[Math.floor(Math.random() * colorPalettes.length)];

  const letters =
    alphabetSets[Math.floor(Math.random() * alphabetSets.length)];

  return {
    ...level,
    id: `${level.id}-style-${Date.now()}`,
    pairs: level.pairs.map((pair, index) => ({
      ...pair,
      id: letters[index] || pair.id,
      color: palette[index] || pair.color,
    })),
  };
}

19. Mock API Notes

The mock API is served using Stoplight Prism and Swagger/OpenAPI.

Important note: Prism validates requests and returns mock responses, but it does not behave like a real database. This means POST may return success, but GET may still return static example scores unless a real backend is added.

20. Tradeoffs and Incomplete Areas

  • Mock API persistence: Scores are optimistically added in UI because Prism does not persist submitted scores like a database.
  • Solver scalability: Current solver works for 5x5. Larger boards would need optimization.
  • Procedural generation: Current generation focuses on solvable layouts. More randomized puzzle generation can be added later.
  • Path rendering: Current path visuals use filled cells. A future improvement would be smooth pipe-like rendering.
  • Backend: A real Node.js or Firebase backend could be added for persistent leaderboard storage.

21. Interview Explanation

A good explanation for the architecture:

I separated the app into screens, reusable components, utilities, solver logic, generator logic, and API services. The Board and Cell components handle rendering and interaction, while the solver is independent from React and can be used for validation and hints. API calls are isolated in gameApi.ts, which makes the UI cleaner and easier to debug. Scores are updated optimistically in the UI before being submitted to the mock API.
